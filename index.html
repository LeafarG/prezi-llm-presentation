1958
Algoritmo Aprendizado supervisionado Mark I

Perceptron · Rosenblatt

Frank Rosenblatt, Cornell Aeronautical Laboratory, propõe o primeiro algoritmo que aprende a partir de dados — não apenas executa lógica hard-coded. É a primeira concretização prática do neurônio matemático de McCulloch-Pitts (1943) com ajuste de pesos via regra de Hebb (1949).

A implementação rodava no Mark I Perceptron, um computador custom-built do tamanho de uma sala, dedicado a treinar e classificar padrões visuais. O hardware lia imagens de 20×20 pixels (resolução baixa mas gigante pra época) através de uma matriz de fotocélulas. Com o tempo, o sistema ajustava os pesos das conexões para separar categorias como "esquerda" vs "direita" ou "triângulo" vs "quadrado".

Foi financiado pela US Navy e ganhou as páginas do New York Times com a manchete "New Navy Device Learns By Doing" (1958). A empolgação inicial foi exagerada — só depois ficou claro que um Perceptron de uma camada tem表达能力 limitada (Hinton chamou mais tarde de "primeiro roundtrip da hype cycle").

Por que importa: o Perceptron é o ancestral direto de toda rede neural. Mesmo com limitações, foi a primeira prova empírica de que máquinas podiam aprender padrões por si, sem serem explicitamente programadas.

1969
Funding cut Symbolic AI domina ~1969-1980

Minsky & Papert · 1º AI Winter

Marvin Minsky (MIT, vencedor do Turing Award) e Seymour Papert publicam Perceptrons (MIT Press, 1969) — uma análise matemática devastadora. O resultado central: um Perceptron de uma única camada não consegue resolver o XOR, uma função não-linearmente separável. O livro generaliza a crítica para várias limitações estruturais.

O livro não era maldoso — era matematicamente correto. Mas o efeito em funding bodies foi catastrófico: DARPA, NSF e outros patrocinadores cortaram verbas para pesquisa em redes neurais. O argumento informal era "se Perceptron não resolve XOR, redes neurais nunca vão resolver problemas reais". (Isso era falso — backprop depois mostrou — mas na época ninguém sabia.)

Resultado: ~1969–1980, o "1º AI Winter". Pesquisadores de redes neurais viraram pária. Symbolic AI — programação lógica, expert systems, regras hand-coded — tomou a dianteira. Foi a era de ouro do LISP e do SHRDLU.

Curiosidade: muitos dos pesquisadores punidos nesse período tinham razão sobre o que viria, mas estavam 20 anos adiantados para o hardware.

1986
Algoritmo Gradient descent Redes multilayer

Backpropagation · Rumelhart, Hinton, Williams

Em 1986, David Rumelhart, Geoffrey Hinton e Ronald Williams publicam Learning representations by back-propagating errors na revista Nature. O paper demonstra, empiricamente e teoricamente, como treinar redes neurais com múltiplas camadas usando gradiente descendente + regra da cadeia (chain rule).

Backprop não era uma ideia 100% nova — Seppo Linnainmaa já tinha descrito o algoritmo em 1970 (na sua tese, em Finlândia), e Paul Werbos aplicou ao contexto de redes neurais em 1974. O paper de Rumelhart-Hinton-Williams foi importante porque fez o algoritmo funcionar na prática — virou a referência para qualquer implementação.

Com backprop, redes multilayer podiam finalmente ser treinadas. Isso destravou a capacidade de aprender representações hierárquicas — base de toda a IA moderna. A essa altura, Yann LeCun já aplicava backprop para reconhecimento de dígitos em cheques (Bell Labs), mostrando viabilidade comercial.

Limitação da época: treinar redes profundas (>3 camadas) era difícil por causa de vanishing gradients + compute limitado. A "deep" em deep learning só viria em 2012 com AlexNet + GPU. Mas backprop estabeleceu a base teórica que faltava.

1987
LISP crash Expert systems oversold ~1987-1993

2º AI Winter · LISP machine crash

O 2º AI Winter foi causado por duas forças convergentes no fim dos anos 80:

(1) Colapso do mercado de LISP machines: Symbolics, LMI e outras vendors de hardware LISP-specialized tinham valuation inflada. Quando workstations Unix/sun e PCs comuns ficaram poderosos o suficiente, o diferencial de preço-valor evaporou. Vendas caíram >50% entre 1987 e 1989. Várias fecharam.

(2) Expert systems under-delivered: sistemas como XCON da DEC, MYCIN na medicina, etc., eram vendidos como "substitutos de especialistas humanos", mas na produção explodiam em custo de manutenção e edge cases. Os 5Cs LISP (caros, complexos, instáveis, lentos, ...) viraram piada interna.

Como durou: ~1987–1993. Período mais profundo do "AI gloom". Symbolic AI funding secou. Quem sobreviveu: pesquisadores de redes neurais persistentes (Hinton, LeCun, Bengio) e o grupo de statistical learning (Vapnik — SVMs).

Curiosidade: Vapnik publicou The Nature of Statistical Learning Theory em 1995 — SVMs eram o estado da arte. Levou 12 anos até a próxima revolta (2012 AlexNet).

2012
CNN GPU ImageNet 60M params

AlexNet · ImageNet moment

Alex Krizhevsky, Ilya Sutskever e Geoffrey Hinton (University of Toronto) submetem uma CNN de 8 camadas e 60 milhões de parâmetros ao desafio ImageNet 2012 (classificar 1.2M de imagens em 1.000 categorias). O resultado é uma chuva de margem: top-5 error de 15,3% contra 26,2% do segundo lugar (uma diferença ridícula).

O twist técnico: treinaram em 2 GPUs NVIDIA GTX 580 ao longo de ~1 semana. GPUs eram hardware de jogo — ninguém tinha pensado em usar para deep learning em escala. Krizhevsky foi pioneiro em CUDA programming.

Outros ingredientes: ReLU activations (em vez de sigmoid/tanh — evita vanishing gradient), dropout (Hinton et al., regularização), e data augmentation. Cada um desses era conhecido, mas a combinação + escala + GPU mudou o jogo.

Efeito cascata: em 6 meses, todo lab sério de visão computacional pivotou pra deep learning. Em 2013, todos os top-10 do ImageNet eram CNNs. Em 2015, ResNet (He et al., Microsoft) atinge top-5 error abaixo de humanos (~5%). A partir daí, "deep learning" virou sinônimo de IA.

Por que importa pra LLMs: a infraestrutura (CUDA + cuDNN + frameworks como Caffe, depois PyTorch/TensorFlow) e os métodos de treinamento (SGD com momentum, dropout, batch norm) que AlexNet popularizou são exatamente o que sustenta o treino de todo LLM moderno.

2017
Attention Parallel Encoder-Decoder

Transformer · "Attention Is All You Need"

Vaswani et al. (Google Brain) publicam em junho de 2017: 8 autores, 12 páginas, "Attention Is All You Need". A proposta: substituir recorrência (RNN/LSTM) e convolução por apenas mecanismo de atenção — uma operação que, a cada passo, pesa o quanto cada token anterior importa para entender o atual.

Três vantagens decisivas:

(1) Paralelização: RNNs processam tokens um de cada vez (sequencial). Transformer processa toda a sequência em paralelo — muitas ordens de grandeza mais rápido pra treinar. Em GPUs, isso é transformador: o tempo de treinamento cai de semanas para dias/horas.

(2) Long-range dependencies: RNNs têm dificuldade de conectar tokens distantes (mesmo com LSTM). Atenção conecta qualquer par de tokens com O(1) hops. Isso destrava contexto longo.

(3) Escala: a arquitetura escala com predictabilidade — mais dados + mais parâmetros = melhor performance (a "scaling law" foi empiricamente caracterizada por Kaplan et al. em 2020, mas a arquitetura já estava lá).

Lineagem: o paper original era encoder-decoder para tradução. Logo decoder-only virou o padrão (GPT-2/3/4, Claude, Llama, Mistral, Qwen, Gemini). Os autores seguiram caminhos variados: Aidan Gomez virou co-founder da Cohere; Llion Jones na Sakana AI; Noam Shazeer fundou Character.AI (depois voltou pra Google).

2020
175B params Few-shot API economy

GPT-3 · 175B parâmetros

Em julho de 2020, a OpenAI lança o GPT-3: 175 bilhões de parâmetros, 96 camadas, ~$4.6M estimados só em compute de treinamento. Foi treinado em ~570GB de texto curado (Common Crawl filtrado + books + Wikipedia).

O headline feature: in-context learning emergiu. Sem nenhum treinamento adicional (gradient updates zero), o modelo consegue aprender uma tarefa nova só com exemplos no prompt — "few-shot prompting". Tradução, sumarização, Q&A, programação básica, tudo "de graça" no prompt. Isso era teoricamente previsto, mas ninguém tinha visto funcionar tão limpo.

A API foi aberta em beta (com waitlist) e virou plataforma. GitHub Copilot (2021), Notion AI, centenas de startups YC — todos construídos em cima de GPT-3 fine-tuned ou prompting. A "AI API economy" explodiu em 2020-21.

Side effects: (i) abriu o debate sobre AI safety/alignment (a OpenAI deliberadamente não publicou os pesos), (ii) lançou o "scale is all you need" narrative que dominou 2020-2022, (iii) motivou a formação do grupo AI Safety do Google Brain (Girshberg et al.), e (iv) mostrou que "capabilities surprise o próprio time de pesquisadores" — efeito que só piorou em modelos seguintes.

2022
GPT-3.5 RLHF 100M usuários / 2 meses

ChatGPT · GPT-3.5 + RLHF

OpenAI lança o ChatGPT em 30 de novembro de 2022. O modelo subjacente era uma variante do GPT-3.5 ajustada com RLHF (Reinforcement Learning from Human Feedback) — anotadores humanos ranqueavam respostas, e um modelo de "recompensa" era treinado nesses rankings; depois, PPO era usado para ajustar o LLM para maximizar a recompensa prevista.

O efeito prático do RLHF: o chatbot seguia instruções, recusava pedidos inadequados com elegância, admitia não saber, e conversava em vez de tentar continuar uma string. Era, finalmente, o que o público leigo imaginava quando pensava em "IA".

Crescimento: 100 milhões de usuários em 2 meses. Recorde absoluto para qualquer app de consumo — TikTok e Instagram levaram ~9 meses. Em janeiro de 2023 a Microsoft anunciou investimento de $10B na OpenAI.

Reações da indústria:

• Google declarou "code red" interno (Sundar Pichai) → acelerou Bard (lançado fev/2023)
• Anthropic (ex-OpenAI) lança Claude (março 2023)
• Meta publica Llama (jul 2023) — open weights
• Google lança Gemini (dez 2023)
• Microsoft 365 Copilot anuncia integração profunda (mar 2023)
• AWS, Salesforce, Adobe — todos anunciam features de IA

Foi "o momento" em que IA deixou de ser tecnologia e virou cultura pop. A corrida começou, e continua até hoje.

2024
o1 reasoning MCP Inference-time compute

Era do raciocínio (o1) + MCP

Em 2024, dois movimentos paralelos redefiniram o ecossistema:

(1) Inference-time compute scaling — OpenAI o1 (set/2024). O modelo "pensa" antes de responder — gera cadeia de raciocínio interna (private tokens) antes da resposta final. Em matemática, código e raciocínio encadeado, isso supera GPT-4 com menos parâmetros. Insight: antes, escalar era só pré-treino. Agora também é inferência — você "paga" com mais latência/custo por respostas melhores. Foi o ano em que "reasoning model" virou categoria.

Outros entrants: DeepSeek-R1 (jan/2025, com pesos abertos, desempenhando similar ao o1), Claude extended thinking, Gemini Thinking.

(2) MCP — Model Context Protocol (Anthropic, nov/2024). Padrão aberto para LLMs se conectarem a ferramentas/dados. Inspirado no USB-C: um cabo, várias fontes. Host (Claude Desktop, Cursor) ↔ Client ↔ Servers (GitHub, Postgres, Google Drive, etc.) via JSON-RPC.

Adoção fulminante: OpenAI Agents, Google DeepMind, Microsoft, Replit, Sourcegraph, Block, Zed, Apollo — todos anunciaram suporte em 2025. Centenas de servers da comunidade. Em 2026, MCP é o padrão de facto pra conectar LLMs a ferramentas — mesmo vendors que têm specs próprios (OpenAI) implementam MCP por interoperabilidade.

2026
Work IQ Cowork Multi-agent

Microsoft Cowork · era agentic

Em 16 de junho de 2026, Microsoft atingiu GA (general availability) do Cowork, seu agente autônomo sobre o ecossistema Microsoft 365. Não é chatbot — é colega de trabalho.

O fundamento técnico é o Work IQ: uma camada de inteligência que lê e-mails, calendário, arquivos, mensagens do Teams, e dados organizacionais, construindo um contexto prático do usuário. Cowork usa Work IQ + LLM + tools para executar tarefas multi-step sem que você especifique cada passo.

Exemplos: "prepara o brief da reunião de amanhã, com base nas últimas decisões do projeto X"; "analisa o pipeline que estagnou, identifica risco, e propõe plano de follow-up"; "compara 4.000 arquivos entre duas versões e resume as diferenças relevantes".

Modelos: Claude Opus e Sonnet da Anthropic + GPT 5.5 da OpenAI — pareados como Advisor (Sonnet, raciocínio de planejamento) e Executor (Opus + GPT, ações). Multi-agent orchestration vira commodity.

Capabilities extras: geração de imagens (DALL-E integration), local browser use (Playwright-equivalente), custom skills/plugins. Aprovações pausam antes de ações sensíveis (enviar e-mail, mover share, criar compromisso).

O que isso sinaliza: a fronteira entre agent e harness dissolveu. Harnesses modernos são agents com UX/governança. A diferença que importa é: harness é o que você compra, agent é o padrão arquitetural. Em 2026, IA deixou de ser ferramenta e virou colega de trabalho.

1956
Conceito McCarthy Dartmouth College

Dartmouth Workshop · nascimento da IA

Verão de 1956, Dartmouth College (Hanover, New Hampshire). John McCarthy (Dartmouth), Marvin Minsky (Harvard), Claude Shannon (Bell Labs) e Nathaniel Rochester (IBM) propuseram um 10-week study sobre inteligência artificial.

A proposta foi notavelmente otimista: um grupo de 10 pesquisadores acreditava resolver boa parte das questões de IA em um único verão. Cobriam redes neurais, computação simbólica, language models primitivos, complexidade, abstração, criatividade. Quase tudo que discutimos hoje estava lá no horizonte.

O workshop não cumpriu a promessa — longe disso. Mas batizou o campo: foi nesse documento que McCarthy cunhou o termo "Artificial Intelligence" (preferido sobre alternativas como cybernetic ou automata theory). A definição hoje clássica de IA vem desse momento.

Curiosidade: entre os participantes estavam Allen Newell e Herbert Simon, que mostraram o Logic Theorist, um programa que provava teoremas de lógica. Foi o primeiro software de IA realmente funcional — meses antes da conferência.

Por que importa: Dartmouth legitimou o campo como disciplina académica financiável. Sem esse impulso, redes neurais e NLP podem ter ficado mais décadas marginalizadas.

1966
Chatbot NLP primitivo Weizenbaum

ELIZA · Weizenbaum (MIT)

Joseph Weizenbaum, do MIT, publica em 1966 o ELIZA, um programa que implementa o script DOCTOR: simula um psicoterapeuta Rogeriano que reformula afirmações do usuário como perguntas.

Exemplo de diálogo:

User: "I'm sad."
ELIZA: "Do you think coming here will help you not to be sad?"
User: "My mother hates me."
ELIZA: "Tell me more about your mother."

Era estatística contextual primitiva: match de keywords + reformulação por templates. Nada de ML, nenhuma rede neural. Mas o resultado impressionou.

Weizenbaum ficou horrorizado quando viu secretárias e pesquisadores do MIT revelarem problemas pessoais reais ao ELIZA, sabendo que era um script. Sua assistente pediu para ele sair da sala quando usou o programa em privado. Isso virou tema central do seu livro Computer Power and Human Reason (1976) — uma crítica à transferência emocional em máquinas.

Por que importa: ELIZA é o ancestral de todo chatbot. O pattern "interface conversacional com personalidade" não morreu — virou ChatGPT, Pi, Character.AI. Toda vez que tu te engajas emocionalmente com um LLM, tá repetindo (em escala) a observação de Weizenbaum.

2025
Open-source Reasoning $5.5M MIT license

DeepSeek-R1 · open-source reasoning

Em janeiro de 2025, a startup chinesa DeepSeek lança o R1: um modelo de raciocínio open-source (MIT license) com performance comparável ao o1 da OpenAI em benchmarks de matemática, código e ciências.

O impacto foi sísmico por três razões:

(1) Custo: o paper técnico reportou ~$5.5M em compute de treinamento. Em contraste, GPT-4 custou estimadamente $100M+, e o1 vendeu API a $200/milhão output tokens. Ordem de grandeza de diferença.

(2) Reasoning explícito: ao contrário de o1 (que esconde o raciocínio interno), R1 mostra tokens de raciocínio diretamente no output. Isso melhorou muito a interpretabilidade do modelo — pesquisadores podem auditar a "cadeia de pensamento". Esse padrão depois foi adotado por Claude, Gemini, GPT-5.

(3) Geopolítica: foi o estopim do "Sputnik moment" da IA. O Nvidia caiu 17% em um único dia (27/jan/2025). Narração: "a China pode treinar modelos frontier mais barato que os EUA". Verdade ou não, o pânico de mercado foi real — e a discussão sobre export controls e cadeias de suprimento de GPUs ficou nacional.

Side effects: a corrida open-source ganhou tração. Em 6 meses, R1-distillados rodavam em laptops (Gemma 3, Llama 3.3, Qwen 2.5-Max). A AI democratization virou tema de novo.

< 10B params
Edge · on-device

SLM · Small Language Models

Modelos com até ~10 bilhões de parâmetros, projetados para rodar em hardware comum: laptop, celular, Raspberry Pi, smart-speakers. A arquitetura é a mesma do Transformer — só que destilada, quantizada ou pré-treinada menor.

Quando usar:

• Privacidade — dados nunca saem do dispositivo (smartphone keyboards, transcription offline).

• Latência — resposta em <100ms, sem round-trip HTTP.

• Custo marginal zero — ótimo para workloads em loop (autocomplete, classificação).

• Edge computing — IoT, automotive, smart home sem cloud.

Trade-offs: capacidade de raciocínio limitada. Não vão escrever Harry Potter. Mas para tarefas estruturadas (extração, classificação, formatação) são surpreendentemente úteis. Phi-3 mini bate GPT-3.5 em vários benchmarks (HellaSwag, ARC).

Players: Microsoft Phi-3 (mini/small/medium), Google Gemma 2B/9B, Meta Llama 3.2 1B/3B, Alibaba Qwen 2.5 0.5B-7B, Apple (silencioso, mas provavelmente tem modelos internos no iOS 18+).

10B – 100B+
General purpose

LLM · o canivete suíço

Modelos de 10 a 100+ bilhões de parâmetros. Grande o suficiente pra tarefas gerais: tradução, sumarização, Q&A, escrita criativa, programação básica. Pequeno o suficiente pra ser self-hosted ou por API a custo aceitável.

Quando usar:

• Produtos B2B — capacidade previsível, custo previsível.

• Multimodal — maioria já fala visão, alguns áudio.

• Janelas longas — até 2M tokens em Llama 3.1; Claude 4 suporta 1M com 'Compaction'.

• Code-assist — Copilot e Cursor rodam LLMs, não Frontier.

Tendência: a fronteira entre LLM "médio" e Frontier "completo" tá esvaziando. Llama 3.1 70B bate GPT-4 original. GPT-4o mini é menor que Claude Sonnet 3.5 mas competitivo.

Players: OpenAI GPT-4o/mini, Anthropic Claude 4 Sonnet, Google Gemini 2 Flash, Meta Llama 3.1 70B/405B, Mistral Large, Microsoft Phi-3 medium.

500B+
Frontier$$$

Frontier · topo da pirâmide

Os modelos mais capazes do mundo em qualquer momento. Hoje GPT-5, Claude 4 Opus, Gemini 2.5 Pro. Estimativas de compute de treinamento: dezenas a centenas de milhões de dólares. Os pesos geralmente não são publicados — só API.

Quando usar:

• Problemas que LLMs não resolvem bem — raciocínio multi-hop, código muito complexo, matemática avançada.

• Workloads pequenas mas críticas — análise jurídica, científico, raciocínio agente.

• Sistemas autônomos — agentes de longa duração, tools chaining.

Custo: GPT-5 cobra ~$10/M input, $30/M output. Opus 4 ~$15/$75. Para 10M tokens/mês de workload agent, $300-$750.

Side effects: a corrida Frontier é capability & safety arms race. Cada nova geração muda capabilities surprises — o time descobre comportamentos não previstos (in-context manipulation, jailbreaks). Por isso as empresas líderes têm red teams internas.

scale at inference
CoTextra compute

Reasoning · pensa antes de responder

Categoria nova que explodiu em 2024. Modelos que alocam compute extra em tempo de inferência para gerar raciocínio intermediário (chain-of-thought, search interno, tentativa-e-erro). OpenAI o1 (set/2024) foi o primeiro produto mainstream.

Mecânica: o modelo emite tokens de raciocínio antes da resposta final. Esses tokens podem ser visíveis (DeepSeek-R1, Claude extended thinking) ou escondidos (o1, o3). Em problemas difíceis, mais raciocínio = melhor resposta (até certo platô).

Quando usar:

• Matemática/programação competitiva — AIME, USACO, Codeforces.

• Raciocínio multi-hop — perguntas que exigem conexão de 5+ fatos.

• Agentes — planners com raciocínio intermediário funcionam melhor.

• Auditoria — quando você quer ver o raciocínio pra entender a falha.

Limitações: latência multi-segundo a multi-minuto. Custo cresce com raciocínio. Em tarefas simples (resumir, formatar) é pior que LLM comum. Use com parcimônia.

Thinking vs No-thinking

A distinção mais importante em 2025-26: modelo que responde direto vs modelo que pensa antes.

No-thinking direto

  • Latência: 100ms – 2s
  • Custo: $0.0001 – $0.01/req
  • Throughput alto, paralelizável
  • Chat, Q&A, sumário, tradução, código simples

GPT-4o · Claude 3.5/4 Sonnet · Gemini 1.5/2.0 · Llama 3.1

Thinking raciocínio

  • Latência: 5s – 60s+
  • Custo: $0.01 – $1+/req
  • Throughput baixo, sequencial
  • Matemática, código complexo, agentes, pesquisa

o1/o3 · DeepSeek-R1 · Claude extended thinking · Gemini Thinking

Ferramentas (Tools)

LLMs são máquinas de texto. Tools dão "mãos" — acesso estruturado ao mundo externo.

Function calling

Modelo recebe schema JSON, retorna tool_call estruturado. Você executa e devolve o resultado.

Structured output

Saída garantida por JSON Schema / Pydantic / Zod. Constrained decoding impede JSON inválido.

Tool use loop

Ciclo: call → result → decide → call → ... → finish. Limites: 5–25 steps, timeout 30–120s.

ReAct pattern

Thought → Action → Observation → repeat. Formalizado em 2022, base da maioria dos agentes.

Web search

Tempo real: Tavily, Brave, SerpAPI, Bing.

Code execution

Sandbox Python/JS. Modelo roda código, instala deps, gera gráficos.

File system

Leitura/escrita/busca. Núcleo de coding agents (Claude Code, Cline).

Database

SQL ou vector search. Text-to-SQL, RAG, retrieval.

Browser automation

Playwright, Puppeteer, Computer Use da Anthropic.

MCP — Model Context Protocol

Padrão aberto criado pela Anthropic (nov/2024). "USB-C para IA." Padroniza a conexão entre LLMs e ferramentas/dados.

[ MCP Host ] ← Claude Desktop · Cursor · Continue │ │ JSON-RPC ▼ [ MCP Client ] ← gerencia 1..N conexões │ ├──▶ [ Server A ] ← GitHub │ ├──▶ [ Server B ] ← Postgres │ └──▶ [ Server C ] ← Google Drive Transport: stdio · HTTP+SSE · streamable HTTP

Capabilities expostas por server MCP:

  • Tools — funções que o modelo chama (similar a function calling)
  • Resources — dados que o modelo lê (arquivos anexados)
  • Prompts — templates reutilizáveis invocáveis pelo usuário
  • Sampling — server pede ao LLM do host para gerar texto

Em 2026, MCP é o padrão de facto para conectar LLMs a ferramentas.
